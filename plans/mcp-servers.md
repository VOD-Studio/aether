## 一、现有基础调研

### 1.1 现有MCP相关接入点与接口定义

当前项目已完成MCP基础框架搭建，核心接入点如下：
| 模块 | 功能 | 状态 |
|------|------|------|
| `src/mcp/config.rs` | MCP配置结构，支持内置工具配置、外部MCP服务器配置（Stdio/HTTP/SSE三种传输类型） | 已完成 |
| `src/mcp/tool_registry.rs` | 统一工具注册表，内置工具注册/查询/执行逻辑，对外暴露`Tool` trait，兼容OpenAI工具格式 | 已完成 |
| `src/ai_service.rs` | 已集成工具调用流程，支持AI自动调用MCP工具，包含工具定义传递、工具执行结果回传逻辑 | 已完成 |
| `src/event_handler.rs` | 已集成MCP配置初始化逻辑 | 已完成 |
| `src/mcp/transport/` | 传输层占位，未实现Stdio/HTTP/SSE传输逻辑 | 待开发 |
| 外部MCP工具管理 | 未实现外部MCP服务器连接、工具自动发现、注册、生命周期管理 | 待开发 |

### 1.2 依赖兼容性评估

#### 现有依赖匹配情况

rmcp 1.1.0 依赖与当前项目依赖完全兼容，无版本冲突：
| 依赖包 | 当前项目版本 | rmcp要求版本 | 兼容性 |
|--------|--------------|--------------|--------|
| async-trait | 0.1 | ^0.1.89 | 兼容 |
| tokio | 1 | ^1 | 兼容 |
| serde | 1 | ^1.0 | 兼容 |
| serde_json | 1 | ^1.0 | 兼容 |
| schemars | 1.2 | ^1.0 | 兼容 |
| reqwest | 0.13 | ^0.13.2 | 兼容 |
| thiserror | 2 | ^2 | 兼容 |
| tracing | 0.1 | ^0.1 | 兼容 |
| url | 2.5 | ^2.4 | 兼容 |

#### 需要新增的rmcp依赖

```toml
# Cargo.toml 新增
rmcp = { version = "1.1.0", features = [
  "client",
  "transport-child-process", # Stdio客户端传输
  "transport-streamable-http-client-reqwest", # HTTP/SSE客户端传输
  "schemars"
] }
```

新增依赖后二进制体积预估增加~2MB，无额外系统依赖。

## 三、分阶段实现方案

### 阶段1：基础传输层实现

#### 产出物

- `src/mcp/transport/stdio.rs` Stdio传输实现
- `src/mcp/transport/http.rs` HTTP/SSE传输实现
- 传输层单元测试

#### 验收标准

- 可以成功连接到Stdio类型的MCP服务器
- 可以成功连接到HTTP/SSE类型的MCP服务器
- 传输错误可以正确捕获并返回
- 单元测试覆盖率 > 90%

#### 预计工时

**2人日**

---

### 阶段2：外部MCP服务器管理

#### 产出物

- `src/mcp/server_manager.rs` MCP服务器管理器
- 服务器连接池、生命周期管理、自动重连逻辑
- 外部工具自动发现与注册逻辑
- 配置热重载支持

#### 验收标准

- 启动时自动连接所有配置的外部MCP服务器
- 服务器断开后自动重试（最多3次，间隔1s/3s/5s）
- 外部MCP服务器的工具自动注册到ToolRegistry
- 工具名称冲突时自动添加服务器名称前缀（如`filesystem__read_file`）
- 支持配置热重载，新增/修改/删除MCP服务器配置无需重启

#### 预计工时

**3人日**

---

### 阶段3：工具调用流程集成

#### 产出物

- 外部工具调用链路打通
- 工具调用权限控制逻辑
- 错误处理与降级逻辑
- 集成测试

#### 验收标准

- AI可以正确识别外部MCP工具并调用
- 工具调用结果正确回传给AI并生成最终响应
- 敏感工具调用需要权限校验
- MCP服务器不可用时自动跳过该工具，不影响正常聊天功能
- 工具调用超时（默认10s）自动返回错误
- 端到端集成测试全部通过

**2人日**

## 四、风险评估与缓解措施

| 风险                            | 影响程度 | 发生概率 | 缓解措施                                                                                              |
| ------------------------------- | -------- | -------- | ----------------------------------------------------------------------------------------------------- |
| 外部MCP服务器不稳定导致请求超时 | 中       | 高       | 1. 配置默认10s超时；2. 失败自动重试1次；3. 连续3次失败自动标记服务器为不可用，30s后再尝试恢复         |
| 工具名称冲突导致调用错误        | 中       | 中       | 1. 注册时自动添加服务器名称前缀；2. 冲突时返回明确错误提示；3. 允许配置自定义名称映射                 |
| 恶意MCP服务器返回危险内容       | 高       | 低       | 1. 仅允许配置在白名单中的MCP服务器；2. 工具返回内容进行敏感词过滤；3. 提供开关可以禁用所有外部MCP工具 |
| 多工具并发调用导致性能下降      | 中       | 中       | 1. 每个MCP服务器配置最大并发数（默认5）；2. 超出并发数时排队执行；3. 提供监控指标查看队列长度         |
| rmcp版本迭代带来兼容性问题      | 低       | 中       | 1. 锁定rmcp主版本号；2. 版本升级前先进行兼容性测试；3. 封装一层抽象层隔离rmcp的API变更                |
